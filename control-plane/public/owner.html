        Quick replies (voice runtime)

        Map caller phrases to one spoken answer. When a transcript contains a phrase (case-insensitive), the receptionist skips the LLM and speaks your reply. Phrases must be at least 4 characters; the first matching intent wins. Requires voice runtime config already published to Redis.

        Receptionist voice

        Choose how your receptionist sounds. Pick a TTS engine first — only controls that apply to that engine are shown. Use the active voice control for preset vs a saved custom voice.

        TTS engine & tuning

        Must match your server’s CHATTERBOX_VARIANT. Turbo is fastest for English but needs a reference voice. Multilingual needs a language code.

        This HTTP API only accepts text, reference audio, and language. No extra per-request sampling controls—tuning is via server env (variant, workers, device).
