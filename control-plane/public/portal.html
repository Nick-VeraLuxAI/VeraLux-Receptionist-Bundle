    Quick replies ▾

    Two paths: Most questions go to the AI (Qwen) using your Greeting & tone context, services & pricing, and transfer contacts — no extra setup. Quick replies here are optional: when a caller’s words contain a phrase you list, the system speaks your fixed answer without calling the model (faster, predictable). Each phrase must be at least 4 characters; first match wins. Requires voice runtime config in Redis.

    Use Add FAQ starters for three local templates from this page. Use Suggest with AI to generate more intents from your text (requires OPENAI_API_KEY on the server). Always review and Save to voice runtime.

    Voice synthesis (TTS) ▾

    Pick a provider — only settings that apply to that engine are shown. Matches the admin voice step.

    Must match your server’s CHATTERBOX_VARIANT. Turbo is fastest for English but needs a reference voice. Multilingual needs a language code. Standard is English without Turbo’s latency focus.

    This HTTP API only accepts text, reference audio, and language. There are no extra per-request “sampling” controls here—tuning is via server env (variant, workers, device).

    Describe how you want it to sound in plain English. Ignored on some small models. Server URL is set above.
